## 摘要
我们提出了Wan-Streamer，一种原生流式、端到端的交互基础模型，从头开始为实时、低延迟、全双工视听交互而设计。Wan-Streamer在单个Transformer内无缝地将语言、音频和视频同时建模为输入和输出，其中序列表示为交错的视觉、音频和文本输入令牌以及视觉、音频和文本输出令牌，通过块因果注意力进行协调以实现增量流式传输。与依赖于独立的VAD、ASR、语言、TTS、音频驱动动画或视频生成模块的级联交互系统不同，Wan-Streamer不依赖外部语言、语音、虚拟形象或视频生成模块：感知、推理、生成、响应时机、话轮管理和跨模态同步都在一个统一模型内联合学习，减少了流水线延迟和错误累积。为了支持自然的视听响应性，我们围绕流式能力重新设计了整个栈，包括因果编码器、因果解码器、块因果注意力和低延迟多模态令牌调度，使得在25 fps下流式单元可短至160毫秒。Wan-Streamer实现了约200毫秒的模型端响应延迟，结合350毫秒的双向网络延迟后总交互延迟约550毫秒，支持亚秒级双工视听通信。这些结果使Wan-Streamer成为用于低延迟流式交互的统一、端到端、多模态交互基础模型。

## Abstract
We present Wan-Streamer, a native-streaming, end-to-end interactive foundation model designed from the ground up for real-time, low-latency, full-duplex audio-visual interaction. Wan-Streamer seamlessly models language, audio, and video as both input and output within a single Transformer, where the sequence is represented as interleaved visual, audio, and text input tokens together with visual, audio, and text output tokens, coordinated by block-causal attention for incremental streaming. Unlike cascaded interactive systems that rely on separate VAD, ASR, language, TTS, audio-driven animation, or video-generation modules, Wan-Streamer does not rely on external language, speech, avatar, or video-generation modules: perception, reasoning, generation, response timing, turn management, and cross-modal synchronization are learned jointly within one unified model, reducing pipeline latency and error accumulation. To support natural audio-visual responsiveness, we redesign the entire stack around streamability, including causal encoders, causal decoders, block-causal attention, and low-latency multimodal token scheduling, enabling streaming units as short as 160 ms at 25 fps. Wan-Streamer achieves approximately 200 ms model-side response latency and approximately 550 ms total interaction latency when combined with 350 ms bidirectional network latency, supporting sub-second duplex audio-visual communication. These results position Wan-Streamer as a unified, end-to-end, multimodal interactive foundation model for low-latency streaming interaction.

### 论文核心总结

#### 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：当前构建实时音视频交互系统的主流方法是使用级联（cascaded）架构，即依次串联独立的语音活动检测（VAD）、自动语音识别（ASR）、大语言模型（LLM）、文本转语音（TTS）、音频驱动动画或视频生成等模块。这种架构存在几个根本性问题：1) **高延迟**：模块间串行处理导致等待时间累积；2) **误差累积**：前一个模块的错误会向后传播并放大；3) **行为不协调**：响应时机、话轮管理、身份保持和长程一致性难以作为整体行为进行学习。
- **整体含义**：论文认为，真正的实时音视频交互本质上是一个全双工（full-duplex）的连续流过程，不是简单的“输入-输出”交替。因此，需要一个**原生流式（native-streaming）** 和**端到端（end-to-end）** 的模型，将感知、推理、生成、同步和交互管理等所有能力在一个统一的框架内联合学习，而不是拼凑多个独立模块。Wan-Streamer 正是为实现这一目标而提出的基础模型。

#### 2. 方法论：核心思想、关键技术细节

- **核心思想**：将交互建模为一个单一的、因果的、连续的令牌流（token stream）。在这个流中，用户的观察（语言、音频、视频）和智能体的响应（语言、音频、视频）一起作为输入和输出，由一个统一的Transformer进行因果建模，所有组件均需满足“可流式处理”的约束。
- **关键技术细节**：
    1.  **统一序列化**：语言、音频和视频的输入与输出令牌被交错排列成一个序列。序列中的每个元素都明确其角色（用户输入或智能体输出）。
    2.  **块因果注意力（Block-Causal Attention）**：设计了一种特殊的注意力机制，确保模型只能看到当前时刻及之前的因果上下文，无法看到未来的信息，从而实现增量式流式生成。
    3.  **完全因果架构**：整个模型的所有组件都被重新设计为满足因果性：
        - **因果VAE（Causal VAEs）**：用于音视频的流式潜在编码。
        - **因果编码器/解码器**：用于因果地处理输入和生成输出。
        - **低延迟令牌调度**：支持在25fps帧率下，流式单元可短至160毫秒。
    4.  **训练目标**：语言响应采用交叉熵损失进行下一令牌预测。音频和视频响应采用条件流匹配（Conditional Flow Matching）损失，在一个联合的扩散Transformer中，同时优化音频和视频的潜在空间的去噪过程。
    5.  **三阶段训练**：
        - **阶段一：独立任务预训练**。分别训练模型的语言理解、多模态理解和音视频生成能力，但将它们混合在一个序列模型中训练，以对齐模态。
        - **阶段二：端到端交互训练**。使用全双工交互数据训练，使模型学会在真实场景下的响应时机、打断、主动聆听和长程一致性。
        - **阶段三：低延迟流式蒸馏**。将更强的教师模型（使用无分类器引导和更多求解步数）蒸馏成学生模型，并采用滚动蒸馏（rolling distillation）策略，让学生模型在其自身生成的历史上进行训练，减少训练与部署时的分布不匹配。
    6.  **推理系统：思考者-执行者（Thinker-Performer）流水线**：模型在推理时被拆分为两个并行的GPU进程。
        - **思考者（Thinker）**：负责处理用户输入（编码）、更新KV缓存、预测语言令牌和运行因果解码器。它执行轻量的“思考”任务。
        - **执行者（Performer）**：仅负责运行昂贵的流匹配求解器，为下一个流式单元生成音频和视频的干净潜变量。
        - 这种设计通过KV缓存交换和任务重叠，将模型端响应延迟降低至约200毫秒，在不牺牲统一模型状态的前提下，最大化硬件利用率。
